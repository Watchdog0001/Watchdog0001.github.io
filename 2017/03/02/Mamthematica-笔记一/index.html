Mathematica 笔记一

Mathematica应该看作编程语言，而不是一个计算软件。Google创始人曾经建议将Mathematica的解释器免费发布，公司盈利则利用 Package 的 License。可惜这个想法没有被采纳，不然也许Mathematica更应用更广泛，我们现在也不至于用盗版的了。目前所知，学习Mathematica一般有三个过程：

  • Procedural programming
    过程式编程，使用 For, While, Do, With等语句，代码具有明显的 C 风格。虽然容易阅读，但效率低，不推荐使用在Mathematica中使用。

  • Functional programming
    函数式编程，目前所知有 Map, Apply, Thread 等函数，效率明显提高。使用一些缩写，就具有Mathematica风格了。
    -Rule-based programming
    Mathematica的精华，感觉太酷了。如随机矩阵的冒泡排序，只要一个语句就够了，RandomReal[1, 10] //. {a, b, c, d} /; b > c -> {a, c, b, d}。

Mathematica核心语言从1.0到11.0都变化不大，所以学习起来不用考虑版本，教材多就都行。以下开始学习：

Everything is an expression

There are two types of expression: atoms and normal expression. Normal expression can be constructed using three blocks: atoms, square brackets and commas. AtomQ[expr] returns True if expr is a atoms. There are three classed of atoms: symbols, numbers and character strings.

Every expression has a head

Function Head[expr] or expr[[0]] gives the head of expr.
