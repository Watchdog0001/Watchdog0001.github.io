Mathematica 笔记一

Mathematica应该看作编程语言，而不是一个计算软件。Google创始人曾经建议将Mathematica的解释器免费发布，公司盈利则利用 Package 的 License。可惜这个想法没有被采纳，不然也许Mathematica更应用更广泛，我们现在也不至于用盗版的了。目前所知，学习Mathematica一般有三个过程：

  • Procedural programming
    过程式编程，使用 For, While, Do, With等语句，代码具有明显的 C 风格。虽然容易阅读，但效率低，不推荐使用在Mathematica中使用。

  • Functional programming
    函数式编程，目前所知有 Map, Apply, Thread 等函数，效率明显提高。使用一些缩写，就具有Mathematica风格了。

  • Rule-based programming
    Mathematica的精华，感觉太酷了。如随机矩阵的冒泡排序，只要一个语句就够了

Mathematica核心语言从1.0到11.0都变化不大，所以学习起来不用考虑版本，教材多就都行。以下开始学习：

Everything is an expression

There are two types of expression: atoms and normal expression. Normal expression can be constructed using three blocks: atoms, square brackets and commas. AtomQ[expr] returns True if expr is a atoms. There are three classed of atoms: symbols, numbers and character strings.

Every expression has a head

Function Head[expr] or expr[[0]] gives the head of expr.

HEXO+Github 搭建个人博客步骤

EXO+Github 教程网上非常多，现将适合自己的步骤记录一下。

安装 Node.js

  • 官网下载 source code，Binaries 试过不行。
  • Ubuntu下，需要root权限安装。 sudo ./configure; sudo make -j4; sudo make install
  • 测试 node -v，显示版本则安装成功。
  • 重启机子，必要。

安装 HEXO

  • sudo npm install -g hexo-cli
  • 初始化 mkdir BLOG; cd BLOG; hexo init . BLOG 为根目录。
  • 生成静态页面 hexo generate; 启动本地服务 bash hexo server
  • 浏览器输入http://localhost:4000, 可以查看博客了。但只能在本地用，下面推送到 Github.

Github

  • 建立Repository, 仓库名必须和用户名一样，格式：your_user_name.github.io。这块卡了好久。
  • 编辑_config.yml文件，来建立关联。注意冒号后面的空格，否则会报缩进不对的错误。

deploy:
type: git
repository: https://github.com/liuzhicheng00/liuzhicheng00.github.io.git
branch: master

  • 推送到远程仓库，然后在浏览器中输入https://liuzhicheng00.github.io/浏览。

npm install hexo-deployer-git –save
hexo generate
hexo deploy

Markdown 语法

慢慢了解即可，可在线编辑，直接看效果，https://www.zybuluo.com/mdeditor#673406。
